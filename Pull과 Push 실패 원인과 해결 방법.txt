Pull과 Push 실패 원인과 해결 방법

1. 증상
로컬 Git과 원격 Git의 버전이 달라서 Pull(원격 -> 로컬)과 Push(로컬 -> 원격)이 실패할 때가 있다.

2. 원인
로컬 Git은 그대로인데 원격 Git이 다른 로컬에 의해 Push되어서 버전이 달라진 경우이가 대부분이다.

3. 해결
(1) 원격 Git의 가장 최신 버전을 확인
ex) c7302a838e751dd0c688958c7f5969efd6477609 [c7302a8]

(2) 로컬 Git의 가장 최신 버전을 확인
ex) 6a0bd70863e74cae74c14f94369bbb379a0e27c5 [6a0bd70]

(3) 서로 다름을 확인하고 로컬의 Git 폴더를 Full Backup
ex) 폴더를 Copy & Paste로 다른 폴더로 옮기거나 구글 드라이브에 올려둠.

(4) 소스트리에서 로컬 탭을 X를 눌러서 삭제.
하지만 여전히 로컬 Git 폴더는 눈에서만 안보이는것이지 실제 삭제된 것이 아니다.

(5) 로컬 Git 폴더를 완전히 삭제.

(6) 소스트리에서 + 버튼을 누르고 Clone 후 소스 경로 / URL에서 깃헙의 주소를 입력.
ex) https://github.com/imhyejeong/OracleStudy.git

(7) 목적지 경로에 로컬 Git 대상이 될 폴더를 생성. (5)번에서 삭제한 폴더명과 동일해도 됨.
ex) C:\dev\DB\ORACLE\OracleStudy

(8) 다시 원격 Git과 로컬 Git의 최종 버전을 비교. 이 때 서로의 버전이 다를 수가 없음.

(9) 로컬과 원격의 버전이 통일되면 로컬에서 원격으로 올릴 즉, (3)번에서 백업해 둔 폴더나 파일을 (7)번 로컬 Git 폴더에 Copy & Paste.

(10) Commit & Push

(11) 로컬 Git의 최종 버전으로 원격 Git의 최종 버전이 수정되었음을 확인.
