■■■ 오라클 설치 및 제거 ■■■

-- 『http://www.oracle.com』	→ 페이지 접근 & 회원가입 	

-- Oracle 18c → 최신 버전 : 유료로 기업에서 쓸 수 있는 버전이라
   Oracle 11g → 설치함. 실무에서는 새로 나온 버전을 바로 적용하기 어려움(유료인 경우가 많음. 어둠의 경로 안됨)

-- eXpress Edition
-- → 법적으로 완전 무료 버전이다.
      기업체나 교육기관 등에서 무료로 사용이 가능한 버전이며,
      프로그램 개방용으로는 충분하지만
      데이터베이스 서버용으로는 다소 부족한 기능을 가진 버전이라 할 수 있다.

-- Standard Edition, Standard Edition One, Data Ware House, Enterprise Edition (가장 상위버전입니다.)	 
-- → 다운로드는 가능하지만, 기업체나 교육기관 등에서 사용하게 되면
      사용 중 검열 시 정식 라이센스를 제시할 수 있어야 한다.
      프로그램 개발용 뿐 아니라, 데이터베이스 서버 용으로도
      충분한 기능을 가지고 있는 버전들이다. 	

-- ○ 기본적으로 설치 과정을 까다롭지 않다.
      (11g eXpress Edition 기준)

	- 기본 설치 경로 : 『C:\oraclexe』
	- SYS 계정 패스워드 설정 : 『java006$』
	- port : 리스너 → 1521 / HTTP리스너 → 8080

-- ※ 참고 
-- 오라클 데이터베이스 파일 위치
-- 오라클 관련 프로그램이 설치되는 경로와 
   관리되고 유지되는 데이터 파일의 위치는 
   물리적으로 다른 경로를 선택하는 것을 권장한다.
   즉, 오라클 관련 프로그램이 C 드라이브에 설치된다고 가정할 때
   데이터 파일의 위치는 D드라이브로 설정하는 것이 바람직하다는 것이다.
   (안정성과 성능 향상)


■■■ 오라클 제거 ■■■

1. 
제어판 > 프로그램 및 기능 → Oracle 관련 항목 제거 

(Window 10 기준)
제어판 > 앱 > 앱 및 기능 → Oracle 관련 항목 제거 : Oracle Database 11g Release 2 Express Edition

2.
실행창 호출(윈도우키 + r) → 『services.msc』입력 

3. 
서비스 리스트의 항목을 통해 확인하면 
『Oracle』 로 시작하는 서비스가 여러개(5개) 확인된다. 
즉, Oracle Server 는 서비스를 기반으로 동작한다는 것이다.
위에서 언급한 것처럼 오라클 프로그램을 제거했다 하더라도
운영체제(OS) 상에서 오라클은 서비스로 동작하기 때문에
이 오라클 서비스를 제거해 주어야 한다.

※ 오라클 서비스 제거하는 방법

	·『윈도우키 + r』키 입력(실행창 호출) → 『regedit』입력(레지스트리 편집기 호출)

	·『HKEY_LOCAL_MACHINE』 > 『SOFTWARE』 > 『ORACLE』 항목 삭제

	·『HKEY_LOCAL_MACHINE』 > 『SYSTEM』 > 『CurrentControlSet』 
	   > 『Services』 > 『Oracle』로 시작하는 모든 항목 삭제 

	·『HKEY_LOCAL_MACHINE』 > 『SYSTEM』 > 『ControlSet001』
 	   > 『Services』 > 『Oracle』로 시작하는 모든 항목 삭제 

	 (오라클을 사용하다보면 DB가 늘어남 
	  → ControlSet002, ControlSet003 까지 생성되므로 모두 지워줘야 함.)

	·『HKEY_LOCAL_MACHINE』 > 『SYSTEM』 > 『ControlSet002』
 	   > 『Services』 > 『Oracle』로 시작하는 모든 항목 삭제 

	·『HKEY_LOCAL_MACHINE』 > 『SYSTEM』 > 『ControlSet002』
 	   > 『Services』 > 『Oracle』로 시작하는 모든 항목 삭제 

	※ 변경된 레지스트리 정보가 적용되지 위해서는
	   반.드.시 재부팅 해주어야 한다.
		
	(변경된 레지스트리 정보는 부팅될 때 반영됨 
  	→ 재부팅 않고 사용시 블루스크린, 먹통 될 수 있음) 

4.
재부팅 이후 탐색기에서 오라큼 홈과 관련된 모든 내용을
물리적으로 삭제한다.

5.
또한 데이터 파일 경로 및 설치 경로의 모든 디렉터리의 파일들을
물리적으로 삭제할 수 있도록 한다.

       (깨끗하게 제거해주지 않으면 다음 Oracle 설치시 문제될 수 있음)
